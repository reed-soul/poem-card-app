## 浏览器控制服务的局限性

**2026-02-11 16:24** - 用户询问为什么不能复用已打开的标签页

**发现的问题：**
- OpenClaw 的浏览器控制服务与用户自己的 Chrome 浏览器是**两个独立的浏览器实例**
- OpenClaw 浏览器：独立的，没有用户的登录状态
- 用户自己的 Chrome：已登录各种账号

**结论：**
要复用用户已登录的浏览器，需要：
1. 安装 OpenClaw Chrome 扩展
2. 在用户浏览器中点击扩展图标来 attach 标签页

**用户反馈：**
- 用户期望直接用自己的浏览器
- 这说明 OpenClaw 的 Chrome 扩展功能可能需要进一步了解

## WebMCP 技术学习

**2026-02-11 18:47** - 用户询问 Philipp Schmid (@_philschmid) 的 WebMCP 推文

**WebMCP (Web Model Context Protocol) 核心概念：**
- 把 MCP (Model Context Protocol) 的理念带到前端
- 让开发者用纯 JavaScript/HTML 暴露网站功能为结构化工具
- 无需单独的后端服务器

**工作原理：**
1. AI agents 访问页面 → 自动发现 WebMCP 工具
2. 从自然语言描述理解工具功能
3. 直接调用工具执行操作
4. 用户保持控制权，可以看到所有操作（Human-in-the-loop）

**实际例子：**
告诉浏览器 agent："找一件 200 元以下的红色夏天连衣裙"
→ agent 直接使用网站自己的搜索/过滤工具，而不是在 UI 中点击

**目标：**
- 开发者：重用现有前端代码，无需新后端框架
- 用户：更快、更可靠的 agent 交互
- 网站：保持对品牌、认证、用户体验的控制

**状态：**
- W3C 社区小组草案（Web Machine Learning CG）
- Chrome 146 提供早期预览（`navigator.modelContext` API）
- GitHub: github.com/webmachinelearning/webmcp

**链接：**
